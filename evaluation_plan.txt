PART 3: EVALUATION METHODS BEYOND LLM-AS-JUDGE
========================================================================

WHY LLM-AS-JUDGE ALONE IS NOT ENOUGH:
  - LLM-as-Judge = another LLM rates your output with no reference answer.
  - It's like grading your own homework.
  - Interviewers will ask: "Who scored it?" → "Another LLM" → skepticism.

EVALUATION TYPES EXPLAINED:

  +---------------------------+---------------------+-------------------------------+-------------+
  | Method                    | Has Expected Answer? | How it works                  | Credibility |
  +---------------------------+---------------------+-------------------------------+-------------+
  | LLM-as-Judge              | No                  | LLM rates output subjectively | Low         |
  | LLM-assisted evaluation   | Yes (ground truth)  | LLM compares output vs YOUR   | Medium-High |
  |                           |                     | answer                        |             |
  | Ground Truth (math only)  | Yes                 | Compute MAE, no LLM needed    | Highest     |
  | Consistency Testing       | N/A                 | Same input 5x, check variance | High        |
  | Pydantic Pass Rate        | Yes (pass/fail)     | Either it parses or it doesn't| High        |
  | Human Evaluation          | Depends             | Human directly rates output   | High        |
  +---------------------------+---------------------+-------------------------------+-------------+

KEY DISTINCTION — "Ground truth + LLM" vs "LLM-as-Judge":

  Pure LLM-as-Judge:
    "Rate this lead score" → LLM says "8/10" (based on what? its vibes)

  LLM-assisted evaluation (ground truth + LLM):
    "Expected score: 75. Agent gave: 82. Is this reasonable?
     Does the scoring breakdown justify the difference?"
    → LLM gives structured reasoning ANCHORED to a reference.

  Ground Truth (math only — most credible):
    Your score: 75 | Agent score: 82 | Error: 7 points
    No LLM needed. Pure math.

WHEN TO USE EACH:

  Use math for (no LLM needed):
    - Score difference (MAE)
    - Pydantic pass/fail
    - Latency measurement
    - Consistency (std dev)

  Use LLM-assisted for (with ground truth):
    - "Is the scoring reasoning logical?"
    - "Does the email sound natural?"
    - "Is the cultural fit analysis justified?"

  Use LLM-as-Judge for (no ground truth available):
    - Open-ended quality assessment
    - Tone/style evaluation
    - Keep it, but never use it ALONE

BEST APPROACH — THREE LAYERS:
  Layer 1: Quantitative (math)    → MAE, Pydantic pass rate, consistency std dev
  Layer 2: LLM-assisted           → Compare agent reasoning vs expected reasoning
  Layer 3: LLM-as-Judge           → Overall quality score (supplementary only)

  Interview answer:
  "I used LLM-as-Judge for qualitative assessment, but I also built ground truth
  evaluation — I manually scored 10 leads and measured a Mean Absolute Error of
  8 points. Pydantic validation passed 95% of the time across 20 runs."


========================================================================
PART 4: EVALUATION METHODS IN DETAIL
========================================================================

1. Ground Truth Comparison (Deterministic)
   - Create 10-15 leads where YOU manually assign the "correct" score.
   - Run the agent, compare its score vs yours.
   - Measure: Mean Absolute Error (MAE), score correlation (Pearson/Spearman).
   - Example: You say lead X should be 75, agent says 82 — that's a 7-point error.
   - Target: MAE < 10 points.

2. Consistency Testing (Stability)
   - Run the same lead through the pipeline 5 times.
   - Measure: standard deviation of scores across runs.
   - If the same lead gets 45, 78, 62, 90, 55 across runs — system is unreliable.
   - Target: standard deviation < 5-10 points per lead.

3. Boundary Testing (Edge Cases)
   - Test leads at the 70-score threshold (the filter cutoff for email generation).
   - Does a lead scoring 71 actually deserve an email? Does a 69 deserve to be skipped?
   - Tests whether the threshold is meaningful and the scoring is calibrated.

4. A/B Email Quality Testing (Human Eval)
   - Generate emails for 10 leads.
   - Have 2-3 people rate them on: personalization, tone, accuracy, would-you-reply.
   - Score on a 1-5 scale, compute inter-rater agreement (Cohen's kappa).
   - More credible than LLM-as-Judge for email quality.

5. Latency & Cost Benchmarking
   - Measure: time per lead, tokens consumed, API cost per lead.
   - Example: "Processes each lead in 55s at $0.03/lead."
   - This is a production metric interviewers care about.

6. Pydantic Validation Pass Rate
   - What % of agent outputs successfully parse into Pydantic models?
   - If 20% of runs fail validation, prompts need work.
   - Target: 95%+ pass rate across 20+ runs.
